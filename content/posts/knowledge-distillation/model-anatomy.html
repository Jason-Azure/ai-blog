什么能通过 API "偷"到？什么不能？（点击查看详情）

📐

架构（Architecture）

Transformer · MoE · Attention

📖 公开
论文公开发表，任何人都可以照着实现。这就像「红烧肉需要酱油和糖」——公共知识，不是秘密。无法通过蒸馏获取（因为根本不需要，直接看论文即可）。
🔥

训练方法（Training Recipe）

学习率 · 课程策略 · 数据配比

🔒 半公开
论文通常公开部分信息，但关键细节故意省略。比如 DeepSeek 论文公开了 GRPO 算法，但训练数据的具体配比、关键超参数等细节并不完整。这些 know-how 是重要竞争力，也无法通过蒸馏获取。
📚

训练数据（Training Data）

万亿 token · 数据筛选 · 质量过滤

🔒 私有
用什么数据、怎么清洗、怎么配比——这是核心商业秘密。Anthropic 的 Project Panama 就是关于训练数据的争议。你可以通过提示注入让模型"吐出"部分训练数据，但无法获取完整数据集。
🎯

对齐数据（Alignment / RLHF）

人类偏好标注 · 安全数据 · 价值观训练

🔒 私有
数千名人类标注员花费数月标注的偏好数据，成本数百万美元。这是让模型从「会说话」变成「说人话」的关键。不公开，也无法通过 API 直接获取。但蒸馏可以间接学到对齐的「效果」（模型的措辞风格、拒绝模式等）。
🧠

权重（Weights）

数十亿~数千亿参数 · 核心资产

🔒 最高
模型的全部「知识」都编码在权重里。通过数月训练、数百万美元算力得到。对于闭源模型，这是最核心的商业秘密。绝对无法通过 API 获取——API 只给你输出文本，不给你内部参数。直接偷权重 = 商业秘密盗窃 = 违法。
⬇ API 调用只能获取 ⬇
📤 输出分布的采样（文本）
只能看到「最后端上桌的菜」
看不到后厨、看不到食材、看不到调料
但这已经包含了大量的「暗知识」
